#Java设计模式及常见问题梳理总结

##单例设计模式总结
知识重点:
    1.私有化构造器
    2.保证线程安全
    3.延迟加载
    4.防止序列化和反序列化破坏单例
    5.防止反射攻击破坏单例

单例的优点:
    1.在内存中只有一个实例,减少了内存开销
    2.可以避免对资源的多重占用
    3.设置全局访问点,严格控制访问

单例的缺点:
    1.没有接口,扩展困难
    2.如果要扩展单例对象,只能修改代码没有其他途径(不符合开闭原则)
